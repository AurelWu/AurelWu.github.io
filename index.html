Introduction

This library makes working with hexagonal grids an intuitive and easy process. It abstracts the math behind a hexagonal coordinate system completely away and provides many useful methods like GetLine, GetRing, GetDistance and many more while accounting for map borders and wrap-around logic.

While it already supports many commonly used map shapes, it can be easily extended to work with more exotic map layouts.
The visual representation of the hexagonal map is not part of the library as that is highly implementation specific. However, there are some sample scenes included which you can use as a starting point for visualizing a hexagonal map. It is structured in a way that you do not need to know the details of hexagonal-grid-coordinate systems, but I still recommend reading Amit Patel's great page about hexagonal grids: https://www.redblobgames.com/grids/hexagons/ at some point.

This package includes examples which will provide a quick introduction to the library. These examples are also described in detail in this documentation


Restrictions

The library has a few restrictions to keep the code and usage as simple as possible. These restrictions will be no issue for the vast majority of projects and for those affected, it should be easy enough to change. (Feel free to contact me at aurel.nosp@m.wuen.nosp@m.sch@g.nosp@m.mail.nosp@m..com) if you need advice regarding those limitations. These are the restrictions:

  • hexagons have a radius of 1 (center to corner) - that concerns only the logic side of things you can still visualize it differently.
  • it only supports Tiles and Edges, not Corners. (we use the term "Tile" for the area/face of the hexagon, edges and corners are hopefully self explanatory)
  • it only works for "pointy-top" hexagons, which are by far more commonly used in computer games than flat-top hexagons .
    left: pointy-top hexagon, right: flat-top hexagon

Different Approaches possible : Generics, creating a child class or keeping the data separate.

The following three approaches are all demonstrated in the included examples.

  1. Using the generic HexMap<Tile> and HexMap<Tile,Edge> classes which allows you full flexibility to use any object as a tile or edge. The HexMap will pass that generic argument through to Tile<T> and Edge<E> who both inherit from a non-generic base class which contains information about the position and index of it.
  2. An alternative approach is to derive from the the non-generic HexMapBase or HexMapBaseWithEdges class and create a concrete implementation.
  3. Finally it is also possible to just use an instance of HexMapBase/HexMapBaseWithEdges and keep all possible Map Data in arrays which are aligned with how the tile and edge positions are indexed.

There are surely even more possibilities so pick the one which suits your needs the most.

Examples

The following examples explain in detail use an approach with generics. Additionally, functionally identical scripts with the 2 other approaches mentioned before are included in the package for the first 3 examples. All these examples use a very simple way to visualise the map which is far from perfect in many ways, especially when it comes to performance. Please keep in mind that this library is not tied to a certain visualisation and you can easily use procedurally generated meshes, Environments created in 3D modelling software or even handpainted backgrounds instead.

These examples are kept as simple and as possible so you can focus on the usage of the library, this code is by no means intended to be used in production code and is not suitable structured for that but for ease of understanding. You will also notice some duplicate code between them, that is consciously done to keep each example self-contained.
